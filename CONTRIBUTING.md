# Guía de Contribución

¡Gracias por tu interés en colaborar con este proyecto! Tu ayuda es fundamental para mejorarlo. 

Para mantener un entorno organizado y eficiente, te pedimos que sigas estas directrices antes de enviar tus aportes.

---

## 📌 Código de Conducta

Al participar en este proyecto, te comprometes a mantener un ambiente respetuoso, inclusivo y libre de acoso. Sé amable y colaborativo con todos los miembros de la comunidad.

---

## 🐛 Cómo Reportar Errores (Bugs)

Si encuentras un error en el código, por favor abre un **Issue** siguiendo estos pasos:

1. **Busca primero:** Asegúrate de que el error no haya sido reportado previamente en la sección de Issues.
2. **Usa un título claro:** Describe brevemente el problema de forma concisa.
3. **Describe el problema:**
   - ¿Qué comportamiento esperabas?
   - ¿Qué comportamiento ocurrió realmente?
   - Pasos detallados para reproducir el error.
   - Capturas de pantalla o mensajes de error si es posible.
   - Versión del software o entorno que estás utilizando.

---

## 💡 Cómo Proponer Mejoras o Funcionalidades

Las sugerencias siempre son bienvenidas. Para proponer una nueva idea:

1. Abre un **Issue** y utiliza la etiqueta (label) de `enhancement` o `feature request`.
2. Explica detalladamente **por qué** esta funcionalidad es necesaria y **cómo** imaginas que debería funcionar.
3. Espera la retroalimentación de los mantenedores antes de comenzar a escribir el código.

---

## 💻 Cómo Enviar tus Cambios (Pull Requests)

Si deseas resolver un problema existente o implementar una mejora aprobada, sigue este flujo de trabajo:

### 1. Preparación del Entorno
* Haz un **Fork** de este repositorio en tu cuenta de GitHub.
* Clona tu fork localmente:  
  `git clone https://github.com`
* Crea una nueva rama (branch) con un nombre descriptivo para tu cambio:  
  `git checkout -b feature/nueva-funcionalidad` o `git checkout -b fix/error-especifico`

### 2. Desarrollo y Reglas de Estilo
* Escribe código limpio, legible y sigue los estándares del proyecto.
* Si el proyecto incluye pruebas (tests), asegúrate de que todas pasen con éxito antes de avanzar.
* Añade nuevas pruebas si estás agregando una funcionalidad.

### 3. Confirmar Cambios (Commits)
* Redacta mensajes de commit claros, en tiempo presente y que describan brevemente lo que hace el cambio.  
  *Ejemplo correcto:* `git commit -m "Corrige el error de validación en el formulario de registro"`

### 4. Crear el Pull Request (PR)
* Sube tus cambios a tu fork en GitHub:  
  `git push origin feature/nueva-funcionalidad`
* Ve al repositorio original y haz clic en **New Pull Request**.
* En la descripción de tu PR, explica qué problemas resuelve y vincula el Issue correspondiente usando palabras clave (ej. `Closes #12`).
* Espera a la revisión del código. Sé paciente; los mantenedores revisarán tu propuesta lo antes posible.
